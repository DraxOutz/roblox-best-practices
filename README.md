## ğŸ‹ï¸ Object Calisthenics in Roblox Lua ##

This repository also follows Object Calisthenics principles, a set of nine rules designed to improve code quality by promoting cohesion, readability, and maintainability in object-oriented programming. These principles are applied in the lessons whenever possible:

- One dot per line: Avoid long chains like player:GetData():GetInventory():GetItem().
- No else keyword: Favor early returns and direct handling of cases.
- Small classes: Each class has a single responsibility.
- Small methods: Methods do one thing only.
- No primitives as members: Wrap values in objects for better encapsulation.
- Collections as objects: Do not expose raw tables; encapsulate them.
- Max 2 attributes per class: Forces cohesive and focused design.
- No getters/setters: Delegate responsibilities to objects themselves.
- Class names under two words: Short, clear, and meaningful.

For more details: [Object Calisthenics Guide](https://developerhandbook.stakater.com/architecture/object-calisthenics.html)
